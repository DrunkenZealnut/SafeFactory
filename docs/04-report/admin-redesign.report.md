### Design (설계 단계)

**문서**: `docs/02-design/features/admin-redesign.design.md`

**설계 결정 사항**:

1. **HTML 구조 전환**
   - `{% extends "base.html" %}` + `{% block title/head_extra/content/scripts %}`
   - 기존 2043줄 독립 HTML → 2019줄 상속 구조

2. **CSS 변수 매핑** (9개 기본 색상)
   - `#f5f7fb` → `var(--sf-bg)` (배경)
   - `#ffffff` → `var(--sf-card-bg)` (카드)
   - `#7c3aed` → `var(--sf-purple)` (브랜드)
   - `#1f2937` → `var(--sf-text-1)` (텍스트 1)
   - `#6b7280` → `var(--sf-text-2)` (텍스트 2)
   - 기타 6개 색상 변수 적용

3. **클래스 접두사** (27개 클래스)
   - `.sidebar` → `.admin-sidebar`
   - `.main-content` → `.admin-main`
   - `.btn` → `.admin-btn`
   - `.card` → `.admin-card`
   - 기타 23개 클래스 일관성 있게 전환

4. **레이아웃 최적화**
   - `max-width: 960px` (프로젝트 컨벤션 준수)
   - sidebar 220px (노트 2: 한글 텍스트 가독성 위해 200px → 220px)

5. **반응형 강화**
   - 기존 768px breakpoint 유지
   - 신규 480px breakpoint 추가 (모바일)

1. **시간 예측**
   - 예상: 3-4시간

   - 교훈: 단순 마이그레이션은 예측이 더 정확 가능

2. **반응형 설계**
   - 480px breakpoint를 사전에 설계에 포함시키는 것이 좋았을 것
   - 모바일 테스트 자동화 추가 필요

3. **색상 시스템**
   - 의미론적 색상(danger/success/warning)은 하드코딩 유지
   - 설계 단계에서 이를 명시적으로 문서화할 필요 있음

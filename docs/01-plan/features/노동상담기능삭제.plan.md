# Plan: 노동상담 기능 삭제 (데이터베이스 유지)

## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | 노동상담(임금/보험/퇴직금/연차/소득세 계산) 기능이 사용률이 낮고, 관련 코드 ~2,500줄이 유지보수 부담을 증가시킴 |
| **Solution** | 계산기 UI/API/서비스 레이어를 제거하되, DB 설정값(SystemSetting 28건)은 보존하여 향후 재활성화 가능성 유지 |
| **Function UX Effect** | 노동법 도메인의 질문 분류·계산 로직이 제거되어 순수 RAG 기반 법률 Q&A로 단순화됨 |
| **Core Value** | 코드베이스 경량화 (~2,500줄 삭제, 파일 10개 제거), 유지보수 범위 축소 |

---

## 1. 배경 및 목적

노동법 도메인에 포함된 임금 계산기, 4대보험 계산기, 퇴직금/연차/소득세 계산 기능을 제거한다.
이 기능들은 독립 모듈(`calculator/`)과 서비스 레이어(`services/labor_calculator.py`, `services/labor_classifier.py`, `services/calculator.py`)로 구성되어 있으며, RAG 파이프라인에도 통합되어 있다.

**제거 이유**: 사용률 대비 코드 복잡도가 높고, 매년 요율 업데이트 유지보수가 필요함.

**데이터베이스 유지 이유**: `SystemSetting` 테이블의 `calc.*` 설정값 28건은 harmless하며, 향후 재활성화 시 재사용 가능.

## 3. 유지 대상

| 항목 | 이유 |
|------|------|
| `SystemSetting` 테이블의 `calc.*` 행 28건 | DB에 남아도 무해, 재활성화 가능 |
| `services/law_api.py` | 법령 검색은 계산과 무관, 유지 |
| `services/rag_pipeline.py`의 law_api 연동 | 법률 Q&A에 필요 |
| `services/rag_pipeline.py`의 legal_analysis 로직 | 법률 분석 패스 유지 |
| `services/domain_config.py`의 laborlaw 도메인 | 노동법 RAG 자체는 유지 |

## 4. 구현 순서

```
1. calculator/ 디렉토리 전체 삭제
2. services/labor_calculator.py 삭제
3. services/labor_classifier.py 삭제
4. services/calculator.py 삭제
5. api/v1/calculator.py 삭제
6. api/v1/__init__.py 수정 (import 제거)
7. api/compat.py 수정 (calculator import + route 제거)
8. web_app.py 수정 (ensure_calculator_settings 제거)
9. models.py 수정 (ensure_calculator_settings 함수 제거)
10. services/rag_pipeline.py 수정 (classifier/calculator 연동 제거, law_api 유지)
```

## 5. 리스크 및 주의사항

| 리스크 | 대응 |
|--------|------|
| rag_pipeline.py 수정 시 law_api 연동까지 실수로 제거 | law_api 관련 코드는 명확히 분리하여 보존 |
| legal_analysis 패스가 classifier에 의존 | classifier 제거 후에도 legal_analysis를 독립적으로 호출하도록 수정 필요 |
| compat route 누락으로 404 발생 | compat.py에서 calculator 관련 import/route 모두 제거 |
| admin 페이지에서 calculator 설정 표시 | DB 데이터는 유지하므로 표시돼도 무해 |

- **DB 변경**: 없음 (의도적 유지)

## Executive Summary

| 항목 | 내용 |
|------|------|
| Feature | my-docs-search-add |
| 작성일 | 2026-03-20 |
| 예상 기간 | 0.5일 |

### Value Delivered

| 관점 | 내용 |
|------|------|
| **Problem** | 현재 "나의 자료" 페이지에서는 이미 북마크한 자료만 볼 수 있고, 새 자료를 추가하려면 학습 페이지(/learn)로 이동하여 검색 후 북마크해야 함 |
| **Solution** | 나의 자료 페이지 내에 검색 기능 추가 → Pinecone 벡터 검색 결과에서 바로 북마크 추가 |
| **Function UX Effect** | 페이지 이동 없이 "나의 자료" 안에서 검색 → 추가 → 관리까지 한 화면에서 완료 |
| **Core Value** | 학습자료 큐레이션 효율 향상 — 탐색과 저장을 분리하지 않음 |

## 1. 현재 구조 (AS-IS)

```
/my-documents 페이지
├── 네임스페이스 필터 (select)
├── 정렬 (최신순/이름순)
├── 북마크 목록 (paginated)
│   ├── 항목 클릭 → PDF 뷰어 or 학습 검색
│   └── 삭제 버튼
└── 일괄 삭제

새 자료 추가하려면: /learn → 검색 → 결과에서 ☆ 클릭 → /my-documents로 돌아옴
```

## 2. 목표 구조 (TO-BE)

```
/my-documents 페이지
├── 🔍 검색 입력 필드 (상단) ← NEW
│   ├── 검색어 입력 → /api/v1/search 호출
│   ├── 검색 결과 표시 (모달 또는 인라인)
│   └── 결과 항목에 [+ 추가] 버튼 → 바로 북마크 생성
├── 네임스페이스 필터
├── 정렬
├── 북마크 목록
│   └── (추가된 항목 즉시 반영)
└── 일괄 삭제
```

---

## 3. 구현 항목

### 3.1 [P1] 검색 UI — 상단에 검색 바 추가
- 검색 입력 필드 + 검색 버튼
- 네임스페이스 선택 (기존 필터와 연동)
- 검색 결과를 현재 목록 위에 별도 영역으로 표시

### 3.2 [P1] 검색 API 호출
- 기존 `/api/v1/search` 엔드포인트 재활용
- 검색 결과에서 `source_file`, `title`(파일명), `namespace` 추출
- 이미 북마크된 항목은 "추가됨" 표시

### 3.3 [P1] 검색 결과에서 바로 북마크 추가
- 각 검색 결과에 [+ 추가] 버튼
- 클릭 → `POST /api/v1/bookmarks` 호출
- 추가 완료 시 버튼 → "✓ 추가됨" 변경
- 북마크 목록 자동 새로고침

## 5. 제외 사항 (YAGNI)
- 검색 결과 미리보기 (PDF 뷰어) — 추가 후 기존 목록에서 열람
- 고급 검색 필터 (날짜, 파일 유형) — 첫 버전에서 불필요
- AI 답변 생성 — 나의 자료 페이지에서는 자료 관리에 집중

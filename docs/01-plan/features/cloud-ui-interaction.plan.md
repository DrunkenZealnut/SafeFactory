### Value Delivered

| 관점 | 내용 |
|------|------|
| **Problem** | 노드 클릭 시 물리 시뮬레이션이 계속 움직여 선택 상태가 불안정하고, 범례 클릭이 즉시 페이지 이동하여 그래프 탐색이 불가능 |
| **Solution** | 선택 시 시뮬레이션 정지 + dimmed 비관련 노드 숨김, 범례 클릭도 그래프 내 하이라이트로 통일 |
| **Function UX Effect** | 클릭 → 관련 네트워크만 정적으로 표시 → 안정적 탐색 경험 |
| **Core Value** | 지식 그래프 본연의 탐색 기능 강화, 학습 흐름 이해도 향상 |

## 1. 요구사항

### 1.1 노드 클릭 시 움직임 정지 + 관련 항목만 표시
- 노드 클릭 시 force 시뮬레이션 **일시 정지**
- 선택된 노드와 **같은 processIdx 그룹**만 하이라이트
- 비관련 노드/엣지는 **거의 투명** (alpha 0.03 수준)
- 같은 노드 재클릭 또는 빈 영역 클릭 시 **시뮬레이션 재개** + 전체 보기 복귀

### 1.2 범례(하단 키워드) 클릭 시 그래프 내 하이라이트
- 범례 클릭 시 `/learn` 페이지 이동 **제거**
- 대신 해당 공정 그룹을 그래프에서 **하이라이트** (노드 클릭과 동일 동작)
- 범례 항목에 **active 스타일** 적용 (선택 상태 시각적 피드백)

## 3. 구현 계획

### 3.1 시뮬레이션 정지/재개
- `selectedNode`가 있을 때 `simulate()` 호출 스킵
- 선택 해제 시 `simulate()` 재개

### 3.2 범례 인터랙션 변경
- `<a href>` → `<span>` 또는 `<button>`으로 변경 (페이지 이동 제거)
- `click` 이벤트에서 `selectedNode`를 해당 공정의 level 1 노드로 설정
- 접근성 목록(`graph-a11y-list`)은 `/learn` 링크 유지 (키보드 사용자용)

### 3.3 시각적 피드백
- 선택된 범례 항목에 active 클래스 → 밝은 텍스트 + 도트 글로우

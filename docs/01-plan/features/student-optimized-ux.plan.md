# Plan: 직업계고 학생 맞춤형 시스템 최적화

## Executive Summary

| 항목 | 내용 |
|------|------|
| Feature | 직업계고 학생 맞춤형 시스템 최적화 |
| 작성일 | 2026-03-19 |
| 예상 기간 | 2~3주 (4개 서브 피처 순차 구현) |
| 우선순위 | 높음 — 실사용자 UX 직결 |

### Value Delivered (4-Perspective)

| 관점 | 내용 |
|------|------|
| **Problem** | 현재 시스템은 전문가 수준의 쿼리를 전제로 설계되어, 직업계고 학생(청소년/초보 노동자)이 구어체로 질문하거나 응급 상황에서 답변 지연이 발생하고, 텍스트 위주의 어려운 답변으로 정보 습득력이 떨어짐 |
| **Solution** | 응급 Fast-Track(Phase 0 이전), 청소년 구어체→전문용어 번역기(Phase 1), 시맨틱 캐싱(Pipeline 전), 학생 눈높이 답변 조절(Phase 7-8) 4가지 계층을 추가 |
| **Function UX Effect** | 응급 질문 1초 이내 응답, 구어체 질문도 정확한 검색 결과, 자주 묻는 질문 즉시 응답, 고1~2 수준 쉬운 답변 + 이미지 병행 제공 |
| **Core Value** | 청소년 노동자의 안전과 권리 보호를 위한 정보 접근성 극대화 — "모르는 것을 모르는" 학생도 필요한 보호를 받을 수 있는 시스템 |

## 1. 배경 및 목적

### 1.1 현재 상황

SafeFactory는 반도체 제조(NCS), 노동법, 안전교육, KOSHA, MSDS 5개 도메인의 RAG 시스템으로, 11단계 파이프라인(P0~P8)을 통해 고품질 답변을 생성한다.

그러나 **실사용자가 '전문가'가 아닌 '직업계고 학생(청소년/초보 노동자)'**이라는 점에서 다음 문제가 존재한다:

| 문제 영역 | 현재 상태 | 영향 |
|-----------|----------|------|
| 응급 상황 대응 | 응급 의도 분류 없음. 모든 질문이 11단계 파이프라인 통과 | 화학물질 사고 등 긴급 상황에서 답변 지연 → 안전 위협 |
| 학생 언어 이해 | 도메인 전문용어 동의어만 존재 (`query_enhancer.py` L125-184) | "약품 냄새 심해요", "알바하다 베였어요" 등 구어체 검색 실패 |
| 응답 속도 | 다중 쿼리 검색 + GraphRAG + 리랭킹 + 외부 API 호출 | 자주 묻는 질문도 매번 전체 파이프라인 실행 → 불필요한 지연 |
| 답변 난이도 | `DOMAIN_PROMPTS`에 난이도 조절 지시 없음 | 전문 용어 가득한 텍스트 답변 → 학생 정보 습득력 저하 |
| 시각적 맥락 | 이미지 검색/표시 기능 존재하나 답변과 분리 | 장비/보호구 설명 시 텍스트만 제공 → 이해도 저하 |

### 1.2 목표

직업계고 학생이 **일상적인 언어로 질문하고, 응급 시 즉각 대응받으며, 쉽고 시각적인 답변**을 받을 수 있도록 시스템을 최적화한다.

### 1.3 성공 지표

| 지표 | 현재 | 목표 |
|------|------|------|
| 응급 질문 응답 시간 | 5~15초 (전체 파이프라인) | < 1초 (Fast-Track) |
| 구어체 질문 검색 성공률 | 측정 불가 (매핑 없음) | > 80% |
| FAQ 질문 응답 시간 | 5~15초 | < 2초 (캐시 히트) |
| 답변 이해도 (가독성) | 전문가 수준 | 고1~2 수준 |

## 2. 스코프

### 2.1 포함 (In-Scope)

#### Sub-Feature A: 응급/긴급 상황 Fast-Track

| 항목 | 내용 |
|------|------|
| 위치 | `run_rag_pipeline()` Phase 0 이전 (Pre-Phase) |
| 핵심 | 응급 키워드 감지 → RAG 우회 → 즉시 응급조치 매뉴얼 스트리밍 |
| 변경 파일 | `services/rag_pipeline.py`, `services/query_router.py`, 신규 `services/emergency_responder.py` |
| 의존성 | 없음 (독립 구현 가능) |

**상세 설계 방향:**
- `EmergencyIntentClassifier`: 키워드 기반 1차 분류 + LLM 2차 확인 (선택적)
- 응급 키워드 사전: "다쳤어", "피가 나", "화학물질 쏟았어", "눈에 들어갔어", "숨이 안 쉬어져", "화상", "감전", "쓰러졌어" 등
- 응급조치 매뉴얼: 도메인별 사전 작성된 정적 응답 (화학물질, 외상, 화상, 감전, 호흡곤란 등 카테고리)
- 응답 포맷: 단계별 넘버링 + 이모지 + 119 신고 안내 필수 포함
- SSE 스트리밍 즉시 시작 (RAG 파이프라인 진입 전)

#### Sub-Feature B: 학생 눈높이 쿼리 번역기

| 항목 | 내용 |
|------|------|
| 위치 | Phase 1 QueryEnhancer 확장 |
| 핵심 | 청소년 구어체 → 산업/노동법 전문 용어 매핑 + 잠재적 위험/법적 보호 추론 |
| 변경 파일 | `src/query_enhancer.py` |
| 의존성 | 없음 (독립 구현 가능) |

**상세 설계 방향:**
- `YOUTH_COLLOQUIAL_MAP`: 구어체/비속어 → 전문 용어 매핑 사전
  - 예: "약품 냄새 심해" → "유해화학물질 노출", "화학물질관리법"
  - 예: "알바하다 베였어" → "산업재해", "근로자 부상", "산재보험"
  - 예: "월급 안 줘" → "임금체불", "근로기준법 위반"
  - 예: "야근 강요" → "연장근로", "청소년 근로시간 제한"
- Multi-Query 프롬프트 강화: "이 질문의 이면에 있는 법적 보호 필요성과 잠재적 위험 요인을 추론하여 검색 쿼리를 확장하세요"
- `expand_with_synonyms()` 내 `YOUTH_COLLOQUIAL_MAP` 통합
- 법적 보호 키워드 자동 추가 로직: 청소년 근로 관련 질문 감지 시 "청소년 근로기준법", "근로계약서" 등 보호 키워드 자동 부스팅

#### Sub-Feature C: 시맨틱 캐싱 계층

| 항목 | 내용 |
|------|------|
| 위치 | `run_rag_pipeline()` 진입 직후 (Phase 0 이전) |
| 핵심 | 의미적으로 유사한 질문의 이전 답변을 캐시에서 즉시 반환 |
| 변경 파일 | 신규 `services/semantic_cache.py`, `services/rag_pipeline.py`, `api/v1/search.py` |
| 의존성 | 임베딩 모델 (기존 `text-embedding-3-small` 활용) |

**상세 설계 방향:**
- 캐시 저장소: SQLite (`instance/semantic_cache.db`) — 별도 Redis 불필요
- 캐시 키: 쿼리 임베딩 벡터 (코사인 유사도 > 0.95 → 캐시 히트)
- 캐시 항목: `{query_embedding, query_text, namespace, response_json, created_at, hit_count, ttl}`
- TTL: 24시간 (FAQ성 질문) / 1시간 (일반 질문)
- 캐시 크기 제한: 최대 1,000개 항목 (LRU 교체)
- 캐시 무효화: 네임스페이스 데이터 변경 시 해당 네임스페이스 캐시 전체 삭제
- 히트율 모니터링: 로깅 및 관리자 패널 통계 표시
- FAQ 사전 시딩: 자주 묻는 질문 목록 사전 캐싱 (SharedQuestion 테이블 활용)

#### Sub-Feature D: 답변 난이도 조절 & 시각적 맥락 강화

| 항목 | 내용 |
|------|------|
| 위치 | Phase 7-8 (컨텍스트 구성 & LLM 생성) |
| 핵심 | 학생 눈높이 프롬프트 + 이미지 참조 통합 |
| 변경 파일 | `services/domain_config.py`, `services/rag_pipeline.py`, `templates/domain.html` |
| 의존성 | 없음 (독립 구현 가능) |

**상세 설계 방향:**

*답변 난이도 조절:*
- `DOMAIN_PROMPTS` 전체에 학생 대상 지시문 추가:
  - "고등학교 1~2학년 수준의 쉬운 어휘를 사용할 것"
  - "친근한 어투(~요/~다) 사용"
  - "전문 용어 사용 시 반드시 괄호 안에 쉬운 설명 병기"
  - "핵심 내용을 먼저 말하고, 상세 설명을 나중에"
  - "단계별 번호 매기기로 구조화"
- 향후 확장: 사용자 프로필 기반 난이도 동적 조절 (v2)

*시각적 맥락 강화:*
- Phase 7에서 수집된 이미지 메타데이터를 LLM 컨텍스트에 포함
  - 예: `[관련 이미지: 방독면 착용법.jpg - 사용자에게 이미지가 함께 표시됩니다]`
- LLM이 이미지 존재를 인지하고 "아래 이미지를 참고하세요" 등 자연스러운 연결
- 프론트엔드: 답변 텍스트 내 이미지 참조 위치에 인라인 이미지 썸네일 표시

## 3. 구현 순서 및 우선순위

```
[A] 응급 Fast-Track ──→ [B] 쿼리 번역기 ──→ [D] 답변 난이도 ──→ [C] 시맨틱 캐싱
     (안전 최우선)          (검색 품질)          (UX 개선)           (성능 최적화)
     ~3일                   ~4일                 ~3일                ~5일
```

### 우선순위 근거

| 순위 | Sub-Feature | 근거 |
|------|-------------|------|
| 1 | A. 응급 Fast-Track | **안전 직결** — 응급 상황 답변 지연은 생명과 연결 |
| 2 | B. 쿼리 번역기 | **검색 품질** — 학생이 질문 자체를 못하면 모든 기능이 무용 |
| 3 | D. 답변 난이도 | **UX 개선** — 검색이 되어도 이해 못하면 의미 없음 |
| 4 | C. 시맨틱 캐싱 | **성능 최적화** — A~D 구현 후 반복 질문 최적화 |

## 4. 기술적 고려사항

### 4.1 기존 코드 영향 분석

| 변경 대상 | 영향도 | 비고 |
|-----------|--------|------|
| `services/rag_pipeline.py` | 높음 | Pre-Phase 분기 추가 (A, C), Phase 7-8 수정 (D) |
| `services/query_router.py` | 중간 | `QUERY_TYPE_CONFIG`에 `emergency` 타입 추가 (A) |
| `src/query_enhancer.py` | 중간 | `YOUTH_COLLOQUIAL_MAP` 추가, 프롬프트 수정 (B) |
| `services/domain_config.py` | 낮음 | `DOMAIN_PROMPTS` 텍스트 수정 (D) |
| `api/v1/search.py` | 중간 | 캐시 히트 처리 분기 (C), 응급 응답 분기 (A) |
| `templates/domain.html` | 낮음 | 인라인 이미지 렌더링 (D) |

### 4.2 리스크

| 리스크 | 확률 | 영향 | 대응 |
|--------|------|------|------|
| 응급 키워드 과감지 (false positive) | 중 | 중 | 2단계 분류: 키워드 1차 → LLM 확인 2차 (선택적) |
| 구어체 매핑 사전 불완전 | 높 | 중 | SharedQuestion 데이터 분석으로 실제 학생 질문 패턴 수집, 점진적 확장 |
| 캐시 무효화 누락 | 낮 | 중 | 네임스페이스 변경 이벤트에 캐시 삭제 훅 연결 |
| 프롬프트 변경으로 기존 답변 품질 저하 | 중 | 높 | A/B 테스트: 기존 프롬프트 vs 학생용 프롬프트 품질 비교 |

### 4.3 의존성

- 외부 의존성 없음 (기존 라이브러리만 사용)
- 임베딩 모델: 기존 `text-embedding-3-small` 활용 (시맨틱 캐시)
- SQLite: 기존 `instance/` 디렉터리 활용
- 응급조치 매뉴얼: 콘텐츠 작성 필요 (KOSHA 가이드라인 기반)

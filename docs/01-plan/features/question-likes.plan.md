# Plan: 질문 좋아요 기능 (Question Likes)

> **Feature**: question-likes
> **Date**: 2026-03-11
> **Project**: SafeFactory

---

## Executive Summary

| Perspective | Description |
|-------------|-------------|
| **Problem** | 사용자들이 AI에게 좋은 질문을 해도 다른 사용자와 공유되지 않아 동일한 질문이 반복되고, 도메인별 "좋은 질문"이 축적되지 않음 |
| **Solution** | AI 답변 완료 후 질문을 공개 공유하는 기능 + 다른 사용자가 좋아요를 눌러 우수 질문을 발굴하는 시스템 |
| **Function UX Effect** | (1) AI 답변 후 "질문 공유" 버튼 → 공개 (2) 도메인별 인기 질문 목록 (3) 좋아요 토글 (4) 인기 질문 클릭 시 즉시 AI 답변 재생성 |
| **Core Value** | 집단지성으로 도메인별 질문 품질 향상, 신규 사용자 온보딩 가속, 플랫폼 네트워크 효과 창출 |

---

## 1. 문제 정의

### 1.1 현재 상태

- 사용자가 AI에게 질문하면 `SearchHistory` 테이블에 **개인 기록**으로만 저장됨
- 다른 사용자가 어떤 질문을 했는지 볼 수 없음
- 도메인별로 "자주 묻는 좋은 질문"이 축적되지 않음
- 샘플 질문은 `domain_config.py`에 하드코딩되어 있어 정적임

### 1.2 해결해야 할 문제

1. **질문 재사용성 부재**: 좋은 질문이 개인 기록에 묻혀 사라짐
2. **신규 사용자 진입장벽**: 어떤 질문을 해야 좋은 답변을 받는지 모름
3. **사용자 간 상호작용 부재**: 검색/질문 경험이 고립적임

## 2. 기능 범위 (Scope)

### 2.1 In-Scope (Phase 1)

| 기능 | 설명 |
|------|------|
| 질문 공유 | AI 답변 완료 후 "이 질문 공유하기" 버튼으로 질문을 공개 |
| 인기 질문 목록 | 도메인 페이지에 좋아요 순 인기 질문 표시 (샘플 질문 영역 대체/보완) |
| 좋아요 토글 | 로그인 사용자가 공유된 질문에 좋아요/취소 |
| 인기 질문 클릭 | 클릭 시 해당 질문으로 AI 답변 즉시 실행 |
| 내가 공유한 질문 | 마이페이지 또는 별도 영역에서 내가 공유한 질문 목록 |

### 2.2 Out-of-Scope (Phase 2+)

- 질문에 대한 댓글/토론 기능
- 질문 태그/카테고리 분류
- 질문 추천 알고리즘 (ML 기반)
- 질문 수정/편집 기능
- 답변 품질 평가 (답변 좋아요)

## 3. 기술 분석

### 3.1 기존 시스템 활용

| 기존 요소 | 활용 방식 |
|-----------|----------|
| `SearchHistory` 모델 | 공유 시 `query`, `namespace`, `answer_preview` 참조 |
| `PostLike` 패턴 | 좋아요 토글 패턴 재사용 (INSERT + IntegrityError) |
| `domain.html` AI 질문 탭 | 답변 완료 후 공유 버튼 추가, 인기 질문 영역 추가 |
| `domain_config.py` | 도메인별 namespace 매핑 재사용 |

### 3.2 신규 모델

**SharedQuestion**: 공유된 질문

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | Integer, PK | |
| user_id | FK(users.id) | 질문 공유자 |
| query | Text | 질문 내용 |
| namespace | String(100) | 도메인 (semiconductor-v2, laborlaw 등) |
| answer_preview | String(300) | AI 답변 미리보기 |
| like_count | Integer | 좋아요 수 (캐시, 실시간 계산 대신 성능 최적화) |
| is_hidden | Boolean | 관리자 숨김 처리 |
| created_at | DateTime | |

**QuestionLike**: 질문 좋아요

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | Integer, PK | |
| question_id | FK(shared_questions.id) | |
| user_id | FK(users.id) | |
| created_at | DateTime | |
| UniqueConstraint | (question_id, user_id) | 중복 방지 |

### 3.3 API 엔드포인트

| Method | Path | 설명 |
|--------|------|------|
| POST | `/api/v1/questions/share` | 질문 공유 (AI 답변 후) |
| GET | `/api/v1/questions/popular` | 도메인별 인기 질문 목록 |
| POST | `/api/v1/questions/<id>/like` | 좋아요 토글 |
| DELETE | `/api/v1/questions/<id>` | 본인 질문 삭제 |
| GET | `/api/v1/questions/my` | 내가 공유한 질문 목록 |

### 3.4 UI 변경 영역

| 위치 | 변경 내용 |
|------|----------|
| `domain.html` AI 답변 영역 | 답변 완료 후 "질문 공유하기" 버튼 |
| `domain.html` 샘플 질문 영역 | 인기 질문 목록으로 대체/보완 (좋아요 수 표시) |
| `domain.html` 인기 질문 | 좋아요 버튼 (♡/♥) + 좋아요 수 |

## 4. 사용자 스토리

### US-1: 질문 공유
> "AI에게 좋은 질문을 해서 유용한 답변을 받았을 때, 다른 사용자도 이 질문을 볼 수 있도록 공유하고 싶다."

**인수 조건**:
- AI 답변이 완료된 후에만 "공유하기" 버튼 표시
- 공유 시 질문 텍스트 + 네임스페이스 + 답변 미리보기(300자) 저장
- 동일 사용자가 동일 질문을 중복 공유하면 안 됨 (query + user_id 기준)
- 비로그인 사용자에게는 공유 버튼 미표시

### US-2: 인기 질문 조회
> "이 도메인에서 다른 사용자들이 좋아요를 많이 누른 질문을 보고, 클릭하면 바로 AI에게 같은 질문을 하고 싶다."

**인수 조건**:
- 도메인 페이지 접속 시 인기 질문 최대 10개 표시 (like_count DESC)
- 각 질문에 좋아요 수와 공유자 이름 표시
- 클릭 시 해당 질문이 입력란에 채워지고 자동으로 AI 답변 시작
- 도메인 필터링 (해당 도메인 질문만 표시)

### US-3: 좋아요 토글
> "다른 사용자가 공유한 질문이 유용하면 좋아요를 눌러 응원하고, 다시 누르면 취소할 수 있다."

**인수 조건**:
- 로그인 사용자만 좋아요 가능
- 좋아요 토글 (누름/취소) 방식
- 본인이 공유한 질문에도 좋아요 가능
- 좋아요 수 실시간 업데이트 (별도 새로고침 불필요)

### US-4: 내 질문 관리
> "내가 공유한 질문 목록을 보고, 필요 없는 건 삭제하고 싶다."

**인수 조건**:
- 내가 공유한 질문 목록 조회
- 개별 삭제 (관련 좋아요도 CASCADE 삭제)

## 5. 위험 요소

| 위험 | 영향 | 완화 방안 |
|------|------|----------|
| 스팸 질문 공유 | 인기 질문 목록 오염 | 사용자당 일일 공유 제한 (10개), 관리자 `is_hidden` 플래그 |
| 부적절한 질문 공유 | 커뮤니티 품질 저하 | 관리자 숨김 처리, 신고 기능 (Phase 2) |
| like_count 캐시 정합성 | 좋아요 수 불일치 | PostLike 패턴(INSERT+IntegrityError) 사용으로 race condition 방지, 주기적 카운트 동기화 |
| 동일 질문 중복 공유 (다른 사용자) | 유사 질문 난립 | Phase 1에서는 허용, Phase 2에서 유사도 기반 중복 감지 검토 |

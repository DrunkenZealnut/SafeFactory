## Executive Summary

| 항목 | 내용 |
|------|------|
| Feature | ncs-category-browser |
| 작성일 | 2026-03-21 |
| 예상 기간 | 1일 |

### Value Delivered

| 관점 | 내용 |
|------|------|
| **Problem** | NCS 학습자료가 95개+ 모듈로 구성되어 있지만, 검색으로만 접근 가능하여 전체 구조를 파악하기 어려움 |
| **Solution** | 학습 페이지에 NCS 4대 카테고리(개발/장비/재료/제조) 트리 브라우저 추가 → 카테고리 선택 후 모듈 목록 표시 → 클릭 시 해당 모듈 검색 |
| **Function UX Effect** | 검색 + 탐색 두 가지 경로로 학습자료 접근 — "뭘 검색해야 할지 모를 때"도 자료 발견 가능 |
| **Core Value** | NCS 학습 체계(능력단위)를 시각적으로 이해하고 체계적 학습 가능 |

## 1. 현재 구조 (AS-IS)

```text
/learn 페이지
├── 검색 입력 → Pinecone 벡터 검색 → 결과 표시
├── 예시 질문 버튼
└── (카테고리 브라우징 없음)

NCS 데이터:
├── 반도체개발 (32 모듈)
├── 반도체장비 (22 모듈)
├── 반도체재료 (25 모듈)
└── 반도체제조 (16 모듈)
→ Pinecone 메타데이터에 ncs_category, ncs_code, ncs_document_title 저장됨
```

## 2. 목표 구조 (TO-BE)

```text
/learn 페이지
├── 검색 입력 (기존 유지)
├── 예시 질문 버튼 (기존 유지)
├── 📚 NCS 카테고리 브라우저 ← NEW
│   ├── [반도체개발] [반도체장비] [반도체재료] [반도체제조] 탭/카드
│   ├── 선택한 카테고리의 모듈 목록 (ncs_document_title 표시)
│   └── 모듈 클릭 → 해당 모듈명으로 검색 실행
└── AI 답변 영역 (기존 유지)
```

## 3. 데이터 소스

Pinecone 메타데이터에서 카테고리/모듈 정보를 집계:
- `ncs_category`: 4개 카테고리
- `ncs_document_title`: 모듈명 (능력단위)
- `ncs_code`: LM 코드

### 3.1 새 API 엔드포인트

```
GET /api/v1/ncs/categories
```

응답:
```json
{
  "categories": [
    {
      "name": "반도체개발",
      "modules": [
        {"code": "LM1903060101_23v6", "title": "반도체 소자 설계"},
        {"code": "LM1903060102_23v6", "title": "반도체 공정 설계"},
        ...
      ]
    },
    ...
  ]
}
```

데이터 소스 옵션:
- **A. DB Document 테이블** — metadata_json에서 ncs_category 집계
- **B. Pinecone 직접 쿼리** — 느림, 비효율
- **C. 파일시스템 스캔** — documents/semiconductor/ncs/data/ 디렉토리 스캔
→ **C 선택** (가장 정확하고 빠름, 기존 filetree 서비스 참고)

## 4. 구현 항목

### 4.1 [P1] API — NCS 카테고리/모듈 목록
- `api/v1/ncs_browse.py` 새 모듈
- 파일시스템에서 4개 카테고리 디렉토리 스캔
- 각 카테고리 하위 모듈 폴더명에서 제목 추출
- 결과 캐싱 (60초)

### 4.2 [P1] 프론트엔드 — 카테고리 브라우저 UI
- `domain.html`에 검색 영역 아래 카테고리 섹션 추가
- 4개 카테고리를 탭/카드로 표시
- 탭 클릭 → 모듈 목록 펼침
- 모듈 클릭 → 검색 입력에 모듈명 자동 입력 + 검색 실행

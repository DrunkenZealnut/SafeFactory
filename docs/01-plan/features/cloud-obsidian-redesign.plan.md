## 1. 옵시디언 그래프 뷰 핵심 기능 (참고)

| 기능 | 설명 |
|------|------|
| **호버 하이라이트** | 노드 호버 시 직접 연결된 노드/엣지만 강조, 나머지 fade |
| **클릭 → 열기** | 노드 클릭 시 해당 노트 열기 |
| **드래그** | 개별 노드를 드래그하여 위치 조정 |
| **줌/패닝** | 마우스 휠 줌, 드래그로 캔버스 이동 |
| **연결 수 기반 크기** | 연결이 많은 노드일수록 크게 표시 |
| **Force 물리** | center force(응집), repel force(반발), link force(연결 탄성), link distance(거리) |
| **텍스트 fade** | 줌 레벨에 따라 텍스트 표시/숨김 |
| **색상 그룹** | 노드를 그룹별 색상으로 분류 |

## 2. 현재 구현 vs 목표

| 기능 | 현재 | 목표 |
|------|------|------|
| 호버 하이라이트 | processIdx 기반 그룹 강조 | **직접 연결 노드만** 강조 + 비연결 fade (옵시디언 방식) |
| 노드 드래그 | 불가 | **개별 노드 드래그** (위치 고정 후 놓으면 해제) |
| 줌/패닝 | 불가 | **마우스 휠 줌 + 드래그 패닝** |
| 노드 크기 | level별 고정 (22/14/8/3.5) | **연결 수 + 인기도(heat) 기반** 동적 크기 |
| 텍스트 표시 | 항상 표시 | **줌 레벨에 따라 fade** (작은 노드는 줌인 시만) |
| 클릭 동작 | 선택 토글 (시뮬레이션 정지) | 유지 (현재 동작 유지 — 옵시디언의 "노트 열기"는 우리 맥락에 맞지 않음) |
| Force 물리 | 기본 구현 | 유지 (이미 구현됨) |

## 3. 구현 항목 (우선순위)

### 3.1 [P1] 호버 시 직접 연결 노드만 하이라이트
- 현재: `processIdx` 기반 전체 그룹 강조
- 변경: 호버된 노드의 **parent/children만** 강조, 나머지 강하게 fade
- 선택(클릭) 시에도 동일 적용

### 3.2 [P1] 노드 드래그
- mousedown on node → 드래그 모드 진입
- 드래그 중 해당 노드의 `fixed = true`, 마우스 따라감
- mouseup → `fixed = false` (물리 시뮬레이션에 복귀)
- 드래그 중에는 시뮬레이션 유지 (다른 노드는 계속 움직임)

### 3.3 [P1] 줌/패닝
- 마우스 휠 → 줌 (transform scale, 0.3x ~ 3x)
- 빈 영역 드래그 → 패닝 (transform translate)
- 줌/패닝은 canvas의 2D transform으로 구현

### 3.4 [P2] 연결 수 기반 노드 크기
- `edges` 배열에서 각 노드의 연결 수 계산
- 연결 수 + heat 값 → 노드 반지름 동적 결정
- level별 base size는 유지하되, 연결 수 보너스 추가

### 3.5 [P2] 줌 레벨 기반 텍스트 fade
- 줌 < 1.2 → level 3 텍스트 숨김
- 줌 < 0.7 → level 2 텍스트도 숨김
- 직접 연결 노드 또는 호버 노드는 줌 무관 항상 표시

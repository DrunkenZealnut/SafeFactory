# Plan: 반도체 답변에 관련 안전보건 내용 자동 첨부

> **Feature**: 반도체답변-안전보건-자동첨부
> **Date**: 2026-03-13
> **Status**: Plan

---

## Executive Summary

| 관점 | 설명 |
|------|------|
| **Problem** | 반도체 도메인 답변이 기술 정보만 제공하고, 해당 공정·물질의 안전보건 위험요인(유해물질, 보호구, 안전수칙)은 누락됨. 사용자가 별도로 안전보건 도메인에서 재검색해야 하는 불편. |
| **Solution** | 반도체 RAG 파이프라인에서 답변 생성 시, 쿼리를 `kosha`(안전보건) 네임스페이스에 자동 크로스 검색하여 관련 안전보건 문서를 찾고, LLM 프롬프트에 보조 컨텍스트로 주입. |
| **Function & UX** | 사용자가 "CVD 공정" 질문 → 기존 반도체 기술 답변 + **"관련 안전보건 정보"** 섹션이 자동 첨부 (유해물질, 보호구, 안전수칙, 응급조치). 별도 검색 불필요. |
| **Core Value** | "기술 정보 + 안전 정보"를 하나의 답변에서 통합 제공하여, SafeFactory의 핵심 가치인 **산업 안전과 기술 교육의 융합**을 실현. |

---

## 1. 배경 및 문제

### 1.1 현재 상태

- 반도체 도메인(`semiconductor-v2` 네임스페이스)은 NCS 기술 문서만 검색
- 안전보건 도메인(`kosha` 네임스페이스)은 안전보건공단 자료만 검색
- 두 도메인은 완전히 분리되어 있어, 반도체 공정의 안전 위험 정보가 답변에 포함되지 않음

### 1.2 문제점

| 시나리오 | 현재 | 개선 후 |
|---------|------|---------|
| "CVD 공정 원리는?" | 기술 원리만 설명 | 기술 원리 + **유해가스(SiH4, NH3) 안전수칙, 보호구** |
| "에칭 공정 종류는?" | 습식/건식 에칭 설명 | 에칭 설명 + **화학물질 취급 주의사항, 응급조치** |
| "CMP 공정이란?" | 화학기계연마 설명 | CMP 설명 + **슬러리 안전관리, 보호장비** |
| "이온주입 공정은?" | 이온 가속 원리 설명 | 이온주입 원리 + **방사선 안전, 고전압 위험** |

### 1.3 기존 유사 패턴

`laborlaw` 네임스페이스에서 이미 유사한 크로스 소스 패턴이 존재:
- `run_rag_pipeline()` → RAG 검색 후 **Law API** 별도 호출 → `law_references_formatted` 생성
- `build_llm_prompts()` → `## 관련 법적 근거` 섹션으로 프롬프트에 주입

이 패턴을 반도체 도메인에 적용: RAG 검색 후 **kosha 네임스페이스 크로스 검색** → 안전보건 컨텍스트 주입.

## 2. 목표

### 2.1 핵심 목표

1. 반도체 도메인 답변 생성 시, 관련 안전보건 정보를 자동으로 검색하여 LLM 프롬프트에 주입
2. LLM이 기술 답변 말미에 **"⚠️ 관련 안전보건 정보"** 섹션을 자연스럽게 생성
3. 안전보건 정보가 없거나 관련성이 낮으면 섹션 생략 (노이즈 방지)

### 2.2 비목표 (Not Goals)

- 모든 도메인 간 크로스 검색 일반화 (향후 확장 가능하지만 이번 범위 아님)
- 안전보건 정보를 별도 UI 카드로 분리 (프롬프트 주입으로 충분)
- 반도체 외 다른 도메인(현장실습, MSDS)에 적용 (이번 범위 아님)

## 3. 기술 접근 방식

### 3.1 크로스 도메인 검색 (Cross-Domain Safety Search)

```
기존 RAG 파이프라인 (Phase 1~7)
    ↓
반도체 namespace 검색 완료, context + sources 확보
    ↓
[NEW] Phase 7.5: Safety Cross-Search
    ├── 쿼리를 kosha 네임스페이스에 벡터 검색 (top_k=5)
    ├── 최소 relevance score 필터 (0.3 이상만)
    ├── 상위 결과에서 안전보건 컨텍스트 구성
    └── safety_context 반환
    ↓
Phase 8: LLM 답변 생성
    ├── 기존 system_prompt + user_prompt
    └── [NEW] ## 관련 안전보건 참고자료 섹션 주입
```

### 3.2 프롬프트 주입 전략

`build_llm_prompts()`에서 반도체 도메인일 때 safety context를 주입:

```
## 관련 안전보건 참고자료
[S1] (출처: 안전보건공단/화학물질_취급_안전수칙.md)
CVD 공정에서 사용되는 SiH4(실란)은 자연발화성 가스로...

[S2] (출처: 안전보건공단/반도체_유해인자_관리.md)
반도체 공정에서 발생하는 유해인자...

위 안전보건 자료를 참고하여, 답변 말미에 "⚠️ 관련 안전보건 정보" 섹션을
추가하세요. 관련성이 낮으면 생략하세요.
```

### 3.3 관련성 판단

모든 반도체 질문에 안전 정보를 붙이면 노이즈가 됨. 필터링 기준:

| 기준 | 값 | 이유 |
|------|-----|------|
| 최소 relevance score | 0.3 | 낮은 관련성 결과 제외 |
| 최대 결과 수 | 3개 | 토큰 예산 제한 |
| 최소 결과 수 | 1개 | 1개도 안 나오면 섹션 자체 생략 |

## 4. 영향 범위

### 4.1 수정 파일 (예상)

| 파일 | 변경 내용 |
|------|----------|
| `services/rag_pipeline.py` | Phase 7.5 크로스 검색 함수 추가, `run_rag_pipeline()` 결과에 `safety_context` 포함 |
| `services/rag_pipeline.py` | `build_llm_prompts()`에 `safety_references` 파라미터 추가 및 프롬프트 주입 |
| `services/domain_config.py` | 반도체 시스템 프롬프트에 안전보건 섹션 활용 지침 추가 |
| `api/v1/search.py` | `build_llm_messages()` 호출 시 `safety_references` 전달 |

### 4.2 성능 영향

| 항목 | 현재 | 변경 후 | 허용? |
|------|------|---------|------|
| 반도체 답변 레이턴시 | ~2-4초 | +200-500ms (벡터 검색 1회 추가) | 허용 |
| LLM 입력 토큰 | ~2000-4000 | +500-1500 (안전 컨텍스트) | 허용 |
| Pinecone API 호출 | 1회 | 2회 (반도체 + kosha) | 허용 |

크로스 검색은 메인 RAG 파이프라인 완료 후 실행하므로, **실패해도 기존 답변에 영향 없음** (graceful degradation).

## 5. 위험 요소

| 위험 | 영향 | 완화 |
|------|------|------|
| 안전보건 결과 관련성 낮음 | 노이즈 답변 | 0.3 이상 score 필터 + "관련성 낮으면 생략" 프롬프트 지시 |
| 레이턴시 증가 | 사용자 대기 | 비동기 검색(메인 검색과 병렬화 가능) |
| kosha 네임스페이스 비어있음 | 기능 무의미 | 결과 0건이면 자동 스킵 |
| LLM 토큰 초과 | 답변 잘림 | 안전 컨텍스트 max 1500자 제한 |
| SSE 스트리밍 호환 | 기존 스트리밍 깨짐 | 프롬프트 주입만이므로 스트리밍 영향 없음 |

## 6. 성공 기준

| 지표 | 기준 |
|------|------|
| 안전보건 섹션 첨부율 | 반도체 질문 중 공정/물질 관련 질문의 60% 이상에서 안전 정보 첨부 |
| 관련성 정확도 | 첨부된 안전 정보가 질문 주제와 실제 관련있는 비율 80% 이상 |
| 레이턴시 증가 | 기존 대비 500ms 이내 |
| 기존 답변 품질 | 기존 반도체 기술 답변 품질 저하 없음 |

## 7. 구현 우선순위

```

P2: 반도체 시스템 프롬프트 안전 섹션 지침 추가
```

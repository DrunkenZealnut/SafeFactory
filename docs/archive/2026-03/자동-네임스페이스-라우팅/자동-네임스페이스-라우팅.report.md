## 6. Lessons Learned & Retrospective

### 6.1 What Went Well (Keep)

- Design 문서의 삭제 대상 상세 (Section 3)가 매우 구체적이어서 누락 없이 8개 파일 수정 완료
- `query_router.py`에 기존 `classify_query_type()` 패턴이 있어 `classify_domain()` 추가가 자연스러웠음
- Plan → Design → Do → Check 순서대로 진행하여 100% Match Rate 달성

### 6.2 What Needs Improvement (Problem)

- Test 6 결과가 Design 예상과 달랐음 — Design 단계에서 키워드 충돌 시나리오를 더 면밀히 분석 필요
- 8개 파일에 걸친 삭제 작업의 검증이 수동 grep에 의존 — 자동화 여지 있음

### 6.3 What to Try Next (Try)

- 도메인 분류 정확도 모니터링: production 로그에서 `[DomainRouter]` 로그 분석하여 오분류 패턴 파악
- 키워드 사전을 별도 JSON/YAML 파일로 분리하면 비개발자도 키워드 추가 가능

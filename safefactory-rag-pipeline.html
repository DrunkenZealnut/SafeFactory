Phase 1Query Enhancement▼
다중 쿼리 생성 + HyDE 가상 문서 + 동의어 확장

Multi-Query

원본 질문에서 의미적 변형 쿼리 2~3개 자동 생성. 다양한 표현으로 검색 커버리지 확대

HyDE

Hypothetical Document Embeddings — LLM이 가상 답변을 생성하고, 그 임베딩으로 검색. 질문-문서 간 의미 갭 해소

동의어 확장

도메인별 동의어 사전으로 쿼리 변형 추가. 예: "식각" → "에칭", "etching"

키워드 추출

BM25 하이브리드 검색용 핵심 키워드 추출. 빈 키워드 시 정규식 폴백 (QW-1)

Phase 5.5MMR Diversity▼
유사 중복 문서 제거로 컨텍스트 다양성 확보

Near-Duplicate Removal

문서 내용의 앞 200자를 기준으로 동일 내용 필터링. 같은 소스에서 추출된 중복 청크가 컨텍스트 슬롯을 낭비하는 것을 방지

Phase 6Context Optimization▼
관련도 필터링 → 중복 제거 → Lost-in-Middle 재배치

스코어 필터링

최소 관련도 점수(MIN_RELEVANCE_SCORE=0.2) 미만 제거. 토큰 수 80 미만 노이즈 청크 제거

Lost-in-Middle 방지

LLM이 중간 문서를 무시하는 현상 방지. 고관련도 문서를 앞뒤에, 저관련도를 중간에 배치

재검색 폴백 (RB-1)

결과 3개 미만 + 도메인 confidence < 0.5 → HyDE 활성화하여 재검색. 기존 결과에 신규 문서 추가

Phase 7Build Context + Sources▼
LLM 프롬프트용 컨텍스트 조립 + 출처 메타데이터 구성

컨텍스트 조립

각 검색 결과를 [N] (출처: file) content 형식으로 번호 매기기. 커뮤니티 글로벌 컨텍스트가 있으면 앞에 삽입

출처 메타데이터

source_file, file_type, score, content_preview, NCS 카테고리, 이미지 URL 등 프론트엔드 표시용 구조화
